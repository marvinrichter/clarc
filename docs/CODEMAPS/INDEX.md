## Key Data Flows

### Session Start
```
SessionStart hook
  ‚Üí scripts/hooks/session-start.js
  ‚Üí detects package manager
  ‚Üí builds SKILL_MAP (all skill names)
  ‚Üí matches glob rules for project type
  ‚Üí injects instinct overlays from ~/.clarc/agent-instincts/
```

### PostToolUse (Edit/Write)
```
PostToolUse: Edit|Write
  ‚Üí post-edit-format-dispatch.js  (auto-format)
  ‚Üí post-edit-workflow-nudge.js   (5 nudges: security, code-review, doc-update,
                                   clarc-self-dev, tdd-guard)
  ‚Üí auto-checkpoint.js            (git checkpoint for /undo)
```

### PostToolUse (Bash)
```
PostToolUse: Bash
  ‚Üí build-failure-router.js       (detect compile errors ‚Üí suggest build-error-resolver)
```

### PreToolUse
```
PreToolUse: Bash
  ‚Üí pre-bash-dispatch.js          (block dev servers, tmux reminder, push review)
PreToolUse: Write
  ‚Üí pre-write-secret-scan.js      (block if secrets in content, exit 2)
  ‚Üí doc-file-warning.js           (warn on non-standard doc paths)
PreToolUse: Edit|Write
  ‚Üí suggest-compact.js            (suggest /compact when context is large)
```

## Hook System Config

Per-project suppression: `.clarc/hooks-config.json`
```json
{
  "disabled": ["code-review-nudge", "tdd-sequence-guard"],
  "code_review_cooldown_minutes": 5,
  "clarc_self_dev_cooldown_minutes": 5,
  "security_scan_cooldown_minutes": 30
}
```

Global config: `~/.clarc/hooks-config.json`
Cooldown state: `~/.clarc/nudge-cooldown.json`

## Clarc Self-Development

When modifying clarc components, the `clarc-self-dev-nudge` (hook 3b in
`scripts/hooks/post-edit-workflow-nudge.js`) fires automatically:

| Modified file | Suggested review agent |
|---------------|----------------------|
| `agents/*.md` | `agent-quality-reviewer` |
| `skills/**/*.md` | `skill-depth-analyzer` |
| `commands/*.md` | `command-auditor` |

See also `CLAUDE.md` ‚Äî "Self-Development Workflow" section.
